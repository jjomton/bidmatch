BidMatch

반복적인 입찰공고 검색을 자동화하고, 기업별 관심 조건에 맞는 공고만 골라 이메일 Digest와 대시보드로 전달하는 업무용 SaaS 프로젝트입니다.

Problem

입찰 담당자의 하루를 잡아먹는 반복 검색을 줄입니다.

공공입찰 담당자는 매일 나라장터에 접속해 키워드, 기관, 지역, 업종, 예산 조건을 반복 입력하고 마감일을 따로 확인해야 합니다. BidMatch는 이 흐름을 수집, 정규화, 매칭, 알림, 운영 로그까지 이어지는 자동화 파이프라인으로 전환합니다.

Core Features

공고를 찾는 일보다 판단하는 일에 집중하게 합니다.

나라장터 자동 수집

용역과 물품 공고를 중심으로 나라장터 OpenAPI를 호출하고, 원본 응답과 정규화 데이터를 함께 저장합니다.

관심 조건 매칭

포함/제외 키워드, 기관, 지역, 업종, 물품분류, 금액 조건을 조합해 사용자의 관심 공고를 선별합니다.

이메일 Digest

매칭된 공고를 사용자별로 묶어 이메일로 발송하고, 발송 성공과 실패 이력을 추적합니다.

마감일 관리

마감 임박 공고와 관심 공고 일정을 대시보드와 캘린더에서 확인할 수 있도록 설계했습니다.

운영 로그

수집 실행 결과, 저장 건수, 오류 메시지, 이메일 발송 상태를 관리자가 확인할 수 있습니다.

AI 요약 확장

Google Gemini API를 활용해 긴 공고 내용을 짧게 요약하는 흐름까지 확장할 수 있습니다.

Workflow

사용자가 조건만 등록하면 나머지는 자동으로 흐릅니다.

01

조건 등록

사용자는 키워드, 기관, 지역, 업종, 물품분류, 금액 조건을 관심 조건으로 저장합니다.

02

공고 수집

Worker가 나라장터 OpenAPI에서 용역과 물품 공고를 수집하고 정규화합니다.

03

자동 매칭

공고와 사용자 조건을 비교하고 제외 키워드와 중복 알림을 걸러냅니다.

04

알림 발송

매칭된 공고를 이메일 Digest로 보내고 발송 이력과 오류 상태를 남깁니다.

Roadmap

MVP에서 운영형 서비스로 확장하는 개발 순서

  1. 회원가입, 로그인, 인증 토큰 검증
  2. 프로필, 기업 정보, 관심 조건 CRUD
  3. 나라장터 API 수집과 Raw 데이터 저장
  4. 공고 목록, 상세, 검색, CSV 다운로드 API
  5. 사용자 조건 기반 매칭과 이메일 Digest 발송
  6. 마감일 캘린더, 관리자 로그, 수동 재조회
  7. AI 요약, 카카오 알림톡, 변경이력 반영
